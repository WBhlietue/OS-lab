\begin{flushleft}
1.1 Үйлдлийн системийн хамгийн чухал 3 зорилго юу вэ?\\
    a)Компьютерийн хэрэглэгчийг компьютерийн hardware программыг ашиглан тохиромжтой, үр ашигтайгаар гүйцэтгэх орчинг бүрдүүлэх.\\
    b)Өгөгдсөн асуудлыг шийдвэрлэхийн тулд шаардлагатай бол компьютерийн нөөцийг тусад нь хуваарилах. Хуваарилалтын үйл явц аль болох шударга, үр ашигтай байх ёстой.\\
    c)Хяналтын программын хувьд энэ нь хоёр үндсэн үүргийг гүйцэтгэдэг: (1) компьютерийг алдаа, зүй бусаар ашиглахаас урьдчилан сэргийлэхийн тулд хэрэглэгчийн програмын ашиглалтанд хяналт тавих\\
\end{flushleft}
\begin{flushleft}
1.2 Computer-н hardware-д үр ашигтай ашиглагдах үйлдлийн систем хэрэгтэйг бид онцолсон. Үйлдлийн систем хэзээ энэ зарчмыг орхиж, нөөцийг "үр ашиггүй " болгох нь тохиромжтой вэ? Яагаад ийм систем үнэхээр үр ашигтай  биш байна вэ?\\
Нэг хэрэглэгчийн систем нь хэрэглэгчийн системийн ашиглалтыг дээд зэргээр нэмэгдүүлэх ёстой. GUI нь CPU-ийн циклийг "хэрэгцээгүй" болгож болох ч хэрэглэгчийн харилцан үйлчлэлийг оновчтой болгодог системтэй.\\
\end{flushleft}
\begin{flushleft}
1.3 Programmer -уудын үйлдлийн системийг бичихэд тулгардаг хамгийн том хүндрэл нь юу вэ?\\
Гол хүндрэл нь үйлдлийн системийг тогтоосон хугацаанд хэрэгжүүлж цаг хугацааг бага зарцуулах явдал юм. Хэрэв систем нь тогтоогдсон хугацаанд  дуусаагүй бол тодорхой хугацааны дараа энэ нь бүхэл бүтэн системийг эвдрэлд хүргэж болзошгүй юм.\\
Үйлдлийн систем нь апп-с гадна веб броусер болон мэйл-н программ гм янз бүрийн тодорхойлолтыг агуулдаг.\\
\end{flushleft}
\begin{flushleft}
1.4 Үйлдлийн систем нь апп-с гадна веб броусер болон мэйл-н программ гм янз бүрийн тодорхойлолтыг агуулдаг. Агуулгыг дэмжих хариултыг өгнө үү.\\
Давуу тал: Өнөөдөр бараг бүх хэрэглэгчид вэб рүү нэвтрэх, цахим шуудангаар ашиглах боломжийг хүсч байгаа бөгөөд энэ зорилгоор вэб хөтөч болон имэйл клиентийг үйлдлийн системтэй хамт багцалсан тохиолдолд хэрэглэгчдэд илүү таатай байх болно. Цаашилбал, вэб хөтчийг (эсвэл бусад програмыг) үйлдлийн системтэй холбох нь  тодорхой давуу талыг бий болгодог. Жишээ нь: Windows linux -г хамтад холбох үед илүү их хурдтай байна\\
\end{flushleft}
\begin{flushleft}
1.5 Kernel mode болон Хэрэглэгчийн mode -н ажиллагааны ялгаа юу вэ? Аюулгүй байдлыг хангаж чаддаг уу?\\
Kernel mode ба хэрэглэгчийн горимын ялгаа нь дараах байдлаар хамгаалалтын анхан шатны хэлбэрийг өгдөг. Зарим зааврыг зөвхөн CPU kernel mode-д байх үед л гүйцэтгэж болно. Үүнтэй адилаар программ kernel mode-д ажиллаж байх үед л техник хангамжийн төхөөрөмжид хандах боломжтой. Тасалдлыг хэзээ идэвхжүүлэх эсвэл идэвхгүй болгохыг хянах нь зөвхөн CPU kernel mode-д байх үед л боломжтой. Тиймээс, CPU нь хэрэглэгчийн горимд ажиллахдаа маш хязгаарлагдмал чадвартай бөгөөд ингэснээр чухал нөөцийн хамгаалалтыг хэрэгжүүлдэг.\\
\end{flushleft}
\begin{flushleft}
1.6 Дараахын аль нь давуу эрхтэй байх ёстой вэ?\\
а. Таймерын утгыг тохируулах.\\
б. Цагийг унш.\\
в. Санах ойг цэвэрлэх.\\
г. Trap заавар гарга.\\
д. Тасалдлыг унтраа.\\
е. Төхөөрөмжийн төлөвийн хүснэгтийн оруулгуудыг өөрчлөх.\\
g. Хэрэглэгчээс цөмийн горим руу шилжих.\\
h. I/O төхөөрөмжид хандах.\\
Дараах үйлдлүүдэд давуу эрх олгох шаардлагатай:\\
Таймерын утгыг тохируулах, санах ойг арилгах, тасалдлыг унтраах, төхөөрөмжийн төлөвийн хүснэгтийн оруулгуудыг өөрчлөх, хандалт хийж\\
I/O төхөөрөмжийг холбох. Үлдсэн хэсгийг хэрэглэгчийн горимд шилжүүлэн гүйцэтгэж  болно.\\

\end{flushleft}
\begin{flushleft}
1.7 Зарим эртний компьютеруудад үйлдлийн системийг суулгаж хамгаалалтыг үүсгэдэг ч зарим санах ойн хуваалт нь үйлдлийн системийг өөрчлөх боломжгүй. Ямар бэрхшээл асуудал үүсэх боломжтой талаар 2 жишээ дээр тайлбар хийнэ үү.\\
Үйлдлийн системд шаардлагатай өгөгдөл (нууц үг, хандалтын удирдлага, нягтлан бодох бүртгэлийн мэдээлэл гэх мэт) нь хамгаалалттайгаар санах ойд хадгалагдах эсвэл дамжих ёстой бөгөөд ингэснээр хэрэглэгчдэд хүртээмжтэй байдаг.\\

\end{flushleft}
\begin{flushleft}
1.8 Зарим CPU нь хоёроос дээш горимоор ажиллах боломжийг олгодог. CPU нь эдгээр олон горимыг ашиглах боломжтой юу?\\
Олон горимыг аюулгүй байдлын нарийн бодлого явуулахад ашиглаж болно.Өөр нэг боломж нь өөр өөр ялгаатай kernel code -р хангах явдал.\\

\end{flushleft}
\begin{flushleft}
1.9 Цаг хэмжигчийг одоогийн цагийг тооцоолоход ашиглаж болно.  Жишээтэй тайлбарлана уу?\\
Программ нь тасалдлыг тооцоолохдоо цагийг ашиглаж болдог. Жишээ нь ирээдүйд унтах цагаа програмын тусламжтай тохируулж болдог. Тасалдлын улмаас одоогийн тасалдлынхаа тоог хянахын тулд local state -г шинэчилдэг. \\

\end{flushleft}
\begin{flushleft}
1.10 Cache яагаад ашигтай байдаг талаар 2 шалтгааныг тайлбарла. Cache ямар асуудлуудыг шийдвэрлэж чадах талаар тайлбар хийнэ үү?\\
Cache нь бүрэлдэхүүн хэсгийн хооронд асуудлуудыг шийдвэрлэнэ. Хэрэь төхөөрөмж хэрэгтэй мэдээллээ олсон бол илүү удаан дахиж хүлээх шаардлагагүй.Cache дэх өгөгдөл заавал байх ёстой бүрэлдэхүүн хэсгийн өгөгдөлтэй нийцэж байх ёстой байдаг.\\

\end{flushleft}
\begin{flushleft}
1.11 Client-server болон Peer-to-peer загваруудын хоорондын ялгааг тайлбарлана уу\\
             Client-server нь үйлчлүүлэгч болон серверийн үүргийг ялгаж өгдөг энэ нь үйлчлүүлэгч серверээс үзүүлж буй үйлчилгээг авахад ч мөн ашиглагддаг.\\
             Peer to peer загварт хатуу үүрэг байдаггүй ба систем нь peer to peer маягаар хийгддэг тул үйлчлүүлэгч аль эсвэл server хоёулангийнх нь үүрэг гүйцэтгэдэг.\\

\end{flushleft}
